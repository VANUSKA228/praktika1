# 🚗 Molar Autoservice — система управления автосервисом

## 📌 О проекте

**Molar Autoservice** — веб-приложение для автоматизации современной станции технического обслуживания. Проект объединяет клиентов, мастеров и менеджеров в единую экосистему, позволяя эффективно управлять заявками на ремонт, вести диагностику автомобилей и планировать рабочие смены персонала.

### Основные возможности

#### Для клиентов:
- Оформление заявок на ремонт через онлайн‑форму.
- Отслеживание статуса выполнения работ (Новая, В работе, Завершена).
- Управление данными о своём автомобиле.

#### Для мастеров:
- Просмотр списка назначенных заявок.
- Ведение чек‑листа диагностики и фиксация повреждений.
- Ввод и хранение OBD‑II кодов ошибок.
- Обновление статуса ремонта в реальном времени.

#### Для менеджеров:
- Управление всеми заказами и клиентской базой.
- Регистрация новых сотрудников (мастеров и менеджеров).
- **Система Schedule:** составление и публикация графиков рабочих смен.
- Контроль прав доступа пользователей.

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/ваш-аккаунт/molar-autoservice.git
cd molar-autoservice
```

### 2. Настройка базы данных

По умолчанию проект использует **SQLite** (не требует установки сервера).  
Если вы хотите использовать PostgreSQL, создайте базу данных и настройте подключение:

```sql
CREATE DATABASE autoservice;
CREATE USER postgres WITH PASSWORD 'ваш_пароль';
GRANT ALL PRIVILEGES ON DATABASE autoservice TO postgres;
```

Настройки БД в `autoservice/settings.py`:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',   # или postgresql
        'NAME': BASE_DIR / 'db.sqlite3',          # для SQLite
        # для PostgreSQL закомментируйте строки выше и раскомментируйте:
        # 'NAME': 'autoservice',
        # 'USER': 'postgres',
        # 'PASSWORD': 'ваш_пароль',
        # 'HOST': 'localhost',
        # 'PORT': '5432',
    }
}
```

### 3. Установка зависимостей и миграции

```bash
pip install -r requirements.txt
python manage.py migrate
```

Если вы переключились с PostgreSQL на SQLite и получили ошибку, удалите файл `db.sqlite3` и выполните `python manage.py migrate` заново.

### 4. Запуск сервера

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)

## 🔒 Безопасность и улучшения

В рамках сопровождения проекта были реализованы следующие мероприятия:

- **Валидация госномера** – проверка формата российского номера при создании заявки (серверная).
- **CSRF‑защита** – удалены все `@csrf_exempt`, добавлен мета‑тег `csrf-token` и заголовки в fetch‑запросах.
- **XSS‑защита** – экранирование пользовательского вывода в шаблонах (фильтр `|escape`).
- **Оптимизация запросов** – исправлена проблема N+1 на странице расписания (`select_related`).
- **Пагинация** – заявки выводятся по 20 на страницу (снижение нагрузки на БД).
- **Миграция данных** – начальные роли (`client`, `master`, `manager`, `admin`) добавлены через миграцию, чтобы избежать ошибки `Role matching query does not exist`.

Все изменения задокументированы в [CHANGELOG.md](CHANGELOG.md).

## 🛠 Решение возможных проблем

| Проблема                        | Причина                                           | Решение                                                                      |
|--------------------------------|---------------------------------------------------|-------------------------------------------------------------------------------|
| `Relation "users" does not exist` | Таблицы не созданы или используется PostgreSQL без миграций | Переключитесь на SQLite или выполните `python manage.py migrate --fake` после развёртывания схемы вручную |
| `Role matching query does not exist` | Отсутствуют записи в таблице `roles`              | Примените миграцию: `python manage.py migrate main 0006` или добавьте роли вручную |
| `403 Forbidden` при регистрации | CSRF‑токен не передан в fetch‑запросе             | Убедитесь, что в шаблоне есть `<meta name="csrf-token">` и заголовок `X-CSRFToken` |
| `Static files not loading`      | `DEBUG = False` без собранной статики             | Для разработки установите `DEBUG = True` или выполните `python manage.py collectstatic` |

|------------|---------------------------|--------------------------------------------|-----------------------|
