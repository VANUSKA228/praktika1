  # 🚗 Molar Autoservice — система управления автосервисом   
## 📌 О проекте   
**Molar Autoservice** — это веб-приложение для автоматизации работы современной станции технического обслуживания. Проект объединяет клиентов, мастеров и менеджеров в единую экосистему, позволяя эффективно управлять заявками на ремонт, вести диагностику автомобилей и планировать рабочие смены персонала.   
### Основные возможности   
### Для клиентов:   
- Оформление заявок на ремонт через онлайн-форму.   
- Отслеживание статуса выполнения работ (Новая, В работе, Завершена).   
- Управление данными о своем автомобиле.   
   
### Для мастеров:   
- Просмотр списка назначенных заявок.   
- Ведение чек-листа диагностики и фиксация повреждений.   
- Ввод и хранение OBD-II кодов ошибок.   
- Обновление статуса ремонта в реальном времени.   
   
### Для менеджеров:   
- Управление всеми заказами и клиентской базой.   
- Регистрация новых сотрудников (мастеров и других менеджеров).   
- **Система Schedule:** составление и публикация графиков рабочих смен.   
- Контроль прав доступа пользователей.

## 🚀 Установка и запуск   
1. **Клонируйте репозиторий:**   
   
Bash   
```
git clone https://github.com/ваш-аккаунт/molar-autoservice.git
cd molar-autoservice


```
1. **Настройте базу данных:**   
   
Создайте БД в PostgreSQL и обновите данные в `settings.py`:   
Python   
```
DATABASES = {
       'default': {
           'ENGINE': 'django.db.backends.postgresql',
           'NAME': 'autoservice',
           'USER': 'postgres',
           'PASSWORD': 'ваша_пароль',
           'HOST': 'localhost',
           'PORT': '5432',
       }
}


```
1. **Примените миграции:**   
   
Bash   
```
python manage.py migrate


```
1. **Запустите локальный сервер:**   
   
Bash   
```
python manage.py runserver


```

## 🛠 Решение возможных проблем   
|                 **Проблема** |                                                **Причина** |                                                                                         **Решение** |
|:-----------------------------|:-----------------------------------------------------------|:----------------------------------------------------------------------------------------------------|
| `Database connection failed` |                              Неверные данные в settings.py |                                                          Проверьте пользователя и пароль PostgreSQL |
|          `Permission Denied` |                            Попытка доступа без нужной роли |                                               Войдите под аккаунтом с правами мастера или менеджера |
|   `Static files not loading` |                           DEBUG = False без сборки статики |                                               Установите `DEBUG = True` в настройках для разработки |

## 📋 API & Маршруты (Основные)   
|      **Метод** |         **Эндпоинт** |                                                   **Описание** |                     **Доступ** |
|:---------------|:---------------------|:---------------------------------------------------------------|:-------------------------------|
|       GET/POST |           `/orders/` |                                     Просмотр и создание заявок |                       Все роли |
|       GET/POST |        `/edit/<id>/` |                                Редактирование заявки и статуса |               Мастер, Менеджер |
|       GET/POST |         `/schedule/` |                                  Просмотр и управление сменами |               Мастер, Менеджер |
|           POST | `/schedule/publish/` |                               Публикация черновиков расписания |                       Менеджер |
|            GET | `/get-car-by-plate/` |                                      Поиск авто по гос. номеру |               Мастер, Менеджер |
|           POST |   `/register-staff/` |                                        Регистрация сотрудников |                       Менеджер |

**Структура репозитория:**   
- `main/` — основная логика (модели, представления, декораторы).   
- `static/` — современные CSS-стили (repair\_m3.css, schedule.css).   
- `templates/` — HTML-шаблоны системы.
